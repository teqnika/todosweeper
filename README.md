# Swipe Todo — 機能仕様書

**ver 1.0 | 2026-03-28**

---

## 1. 概要

Swipe Todo は、Tinder 風のカードスワイプ UI でタスクを直感的に管理できる Web アプリケーション。スワイプ・タップ操作に最適化しており、PC・スマートフォン両対応のレスポンシブ設計。

---

## 2. インフラ構成

### 2.1 推奨構成

| レイヤー | サービス | 備考 |
|---|---|---|
| フロントエンド | Cloudflare Pages | 無制限リクエスト・CDN 配信 |
| バックエンド API | Cloudflare Workers (Hono) | エッジ実行・Cold Start なし |
| データベース | Cloudflare D1 (SQLite) | 5GB 無料・Workers 内で完結 |

### 2.2 選定理由

- 永続無料枠で完全運用可能
- Cloudflare エッジネットワークによる低レイテンシ
- D1 を使うことで外部 DB 接続が不要になりコードが最小化
- `git push` だけで自動デプロイ可能

### 2.3 技術スタック

| 分類 | 技術 |
|---|---|
| フレームワーク | React + Vite (TypeScript) |
| API | Hono on Cloudflare Workers |
| DB | Cloudflare D1 (SQLite) |
| デプロイ | Wrangler CLI / GitHub 連携 |

### 2.4 比較検討（不採用案）

| 構成 | 理由 |
|---|---|
| AWS Lambda + DynamoDB | フロント（S3+CloudFront）が12ヶ月後に有料化。設定が煩雑 |
| GCP Cloud Run + Firestore | Firestore は NoSQL のみ。RDB 不要なら有力候補 |
| Vercel + Neon + Prisma | Cold Start あり（数百ms）。Workers 非対応のため Prisma 使用不可 |
| Fly.io + Neon | Docker 設定必要。コード量が増える |

## 4. コア機能

### 4.1 スワイプ操作

カードを 4 方向にスワイプして各アクションを実行。タッチ（モバイル）・マウスドラッグ（PC）両対応。

| 操作 | アクション | 詳細 |
|---|---|---|
| ← 左スワイプ | ✓ 完了 | タスクを完了済みに移動 |
| → 右スワイプ | → 後回し | スタックの末尾へ移動 |
| ↑ 上スワイプ | 🗑 削除 | 確認モーダル後にゴミ箱へ移動 |
| ↓ 下スワイプ | ★ 優先度UP | 優先度スコア +1・スタック先頭へ昇格 |

> スワイプ判定閾値：80px。8px 未満の移動はタップとして判定し、編集モーダルを開く。

### 4.2 カードタップで編集

- 最前面のカードをタップ（クリック）すると編集モーダルが開く
- スワイプ操作（8px 以上のドラッグ）とタップを自動で区別
- 編集可能項目：メモ・期限
- 背面のカード（2枚目）はタップ無効

### 4.3 アクションボタン

スワイプが難しい環境向けに ✓ / 🗑 / ★ / → の 4 ボタンをカード下部に表示。ボタン押下でスワイプと同等のアクションを実行。

### 4.4 タスク追加

| 方法 | 説明 |
|---|---|
| 単体追加（＋ボタン） | タイトル・メモを入力して 1 件追加 |
| 一括追加（≡＋ボタン） | テキストエリアに 1 行 1 タスクで複数件を一括登録。リアルタイムプレビュー付き |

### 4.5 明日まで非表示（スヌーズ）

- 💤 ボタンを押すと、当該タスクを翌日まで非表示にする
- 期限が自動的に翌日に延長される
- 翌日以降、スタックに再表示される
- ヘッダー ☰ バッジに非表示件数を表示

### 4.6 優先度管理

- 下スワイプで優先度スコア（+1）が加算され、スタック先頭に昇格
- カードに「★ PRIORITY N」バッジを表示

### 4.7 Undo（元に戻す）

- ↩ ボタンで直前のアクションを取り消す
- todos・done・trash すべての状態を履歴管理

## 5. タスク管理

### 5.1 期限管理

| 状態 | 表示 | 色 |
|---|---|---|
| N日超過 | 「N日超過」 | 赤 `#F87171` |
| 当日 | 「今日まで」 | 黄 `#FBBF24` |
| 翌日 | 「明日まで」 | オレンジ `#FB923C` |
| N日後 | 「N日後」 | グレー |

カードおよびタスク一覧の両方に期限バッジを表示。

### 5.2 完了・未完了の切り替え

- 左スワイプ / ✓ ボタン → 完了タブへ移動
- 完了タブの「↩ 戻す」→ 未完了スタックの先頭へ復元
- タスク一覧画面でも同操作が可能

### 5.3 削除・ゴミ箱

| 操作 | 動作 |
|---|---|
| 削除ボタン / 上スワイプ | 確認モーダル表示 → 承認でゴミ箱へ移動 |
| ゴミ箱「↩ 復元」 | 未完了スタックに戻す |
| ゴミ箱「完全削除」 | 完全に削除（復元不可） |
| ゴミ箱を空にする | ゴミ箱内の全件を完全削除 |

> 削除確認モーダルにはタイトル・メモのプレビューを表示し、誤削除を防ぐ。

## 6. タスク一覧画面

ヘッダー ☰ ボタンから遷移。フィルタータブで表示を切り替え。

| タブ | 内容 |
|---|---|
| 全て | 未完了 + 完了の全タスク |
| 未完了 | 未完了タスクのみ |
| 完了 | 完了済みタスクのみ |
| 🗑️ N件 | ゴミ箱内タスク |

### 各行の操作ボタン

| タスク状態 | 表示ボタン |
|---|---|
| 未完了 | 編集 / ✓ 完了 / 🗑 削除 |
| 完了 | ↩ 戻す |
| ゴミ箱 | ↩ 復元 / 完全削除 |

## 7. タスクデータ構造

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | `number` | 一意 ID（`Date.now() + index`） |
| `title` | `string` | タスクタイトル |
| `memo` | `string` | メモ（任意） |
| `priority` | `number` | 優先度スコア（下スワイプで +1） |
| `dueDate` | `string \| null` | 期限日（`YYYY-MM-DD`） |
| `snoozedUntil` | `string \| null` | スヌーズ解除日（`YYYY-MM-DD`） |
| `trashedAt` | `string` | ゴミ箱移動日時（ISO 8601） |

## 8. デザイン仕様

### 8.1 カラーパレット

| 用途 | カラーコード |
|---|---|
| プライマリ（紫） | `#7C3AED` |
| 完了（緑） | `#4ADE80` |
| 削除（赤） | `#F87171` |
| 優先度（黄） | `#FBBF24` |
| スヌーズ（青） | `#60A5FA` |
| 後回し（ラベンダー） | `#A78BFA` |
| 画面背景 | `#1C1C2E` 〜 `#18182A` |
| カード背景 | `#28283E` 〜 `#222236` |

### 8.2 フォント

| 用途 | フォント |
|---|---|
| 見出し・タイトル | Syne（800 weight） |
| 本文・メモ | DM Sans（400 / 500） |
| ラベル・バッジ | DM Mono（400 / 700） |

### 8.3 インタラクション

- スワイプ方向インジケーター：ドラッグ量に応じてオーバーレイが段階的に表示
- カードの回転：X 方向ドラッグ量 × 0.08deg
- ボタンホバー：`scale(1.08)` + 背景色変化
- トースト通知：アクション後に上部にフェードイン表示（1.8 秒）

## 9. 非機能要件

| 項目 | 仕様 |
|---|---|
| 対応デバイス | PC・スマートフォン（レスポンシブ） |
| タッチ操作 | Pointer Events API（タッチ・マウス統合） |
| コスト | 永続無料（Cloudflare Free Tier） |
| Cold Start | なし（Cloudflare Workers エッジ実行） |
| Workers リクエスト上限 | 10 万 req / 日 |
| D1 ストレージ | 5 GB |
| 状態管理 | React `useState`（現時点はクライアント側のみ） |
